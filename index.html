    4. TensorBoard可视化：

      • Tensorboard可以记录与展示以下数据形式：标量 Scalars、图片 Images、音频 Audio、计算图 Graph、数据分布 Distribution、直方图 Histograms、嵌入向量 Embeddings
      • Tensorboard的可视化过程
        • 建立一个graph,想从graph中获取某些数据的信息
        • 确定要在graph中的哪些节点放置summary operations以记录信息
        • 使用 tf.summary.scalar 记录标量
        • 使用 tf.summary.histogram 记录数据的直方图
        • 使用 tf.summary.distribution 记录数据的分布图
        • 使用 tf.summary.image 记录图像数据
        • operations并不会去真的执行计算，除非run,或者它被其他的需要run的operation所依赖。
        • 特地去运行所有的summary节点。一份程序下来可能有超多这样的summary 节点，要手动一个一个去启动极其繁琐，可以使用tf.summary.merge_all去将所有summary节点合并成一个节点，只要这个节点，产生所有之前设置的summary data。
        • 使用tf.summary.FileWriter将运行后输出的数据都保存到本地磁盘
        • 运行整个程序，并在命令行输入运行tensorboard的指令，之后打开web端查看可视化的结果

Notes：

面向对象三大特性：

封装、继承、多态
封装：将对象的属性和行为抽象出来包装到类中；
继承：将具有相同属性和行为的对象抽象出来包装成一个父类；
多态：多种形态，相同的属性和行为不同的表现方式，比如动物叫声；

面向接口

接口是为了处理对象之间的协作关系，将定义与实现分离，实现系统解耦目的
例如为抽象类设计行为模型；

Reflection

1. 反射，用来通过对象获取类

　　实例：
　　import java.lang.reflect.Method
　　Class<?> c = null; //声明对象
　　c = Class.forName(“org.lxh.demo.SimpleOne”); //实例化
　　Method toM = c.getMethod(“toString”); //取得toString方法
　　toM.invoke(c.newInstance()); //invoke()进行调用

  • Class.newInstance()用来反射无参的构造器
  • Constructor.newInstance()可以反射任何构造器

2. invoke()方法：调用包装在当前Method对象中的方法

　　参数（obj：实例化后的对象，args：用于方法调用的参数）
　　有参数的情况：
　　Method met = c.getMethod(“sayHello”,String.Class,int.Class);
　　String rv = null;
　　rv = (String)met.invoke(c.newInstance,”xxk”,21);

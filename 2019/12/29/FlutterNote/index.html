Flutter基础笔记，从板件计算小APP开始的经验整理，最根本的知识理解，打牢基础

Notes

  • Flutter算是支持平台最多的框架（windows、android、ios）
  • Flutter的核心是组件，最基础是widget，其余组件继承StatefulWeidget和StatelessWidget
  • 入口程序void main() => runApp(Widget app);是Dart语言特有的速写形式
  • material.dart是设计风格的基础包，类似网页中按钮、选项的组件库
  • @override Widget build(BuildContext context)重写方法构建Widget，返回值也是widget
  • 设置点击事件，padding会响应，margin不会响应
  • 长列表组件通过列表项构造器itemBuilder实现
  • 获取表单实例：
    • 设置全局类型key，通过其属性获取表单属性
      • GlobalKey<FormState> loginKey = new GlobalKey<FormState>();
    • 读取当前Form状态使用
      • var loginForm = loginKey.currentState;
  • home属性设置应用主页
  • 使用List.generate生成数据List.generate(20 , (i) => new Product('商品$i','这是商品$i'),
  • 导航器Navigator组件管理路由，堆栈原理，并传送和返回数据。这里可以借鉴板件计算项目和参考书上的不同使用方法。
  • 添加手势、设置字体，启动页、组件装饰、Hero实现动画、Flutter插件使用蓝牙、测试开发等操作可以具体参考亢少军老师的《Flutter技术入门与实战》一书

无状态组件和有状态组件

  • 无状态组件的属性是不变的，有状态的在生命周期中发生变化
  • 举例说明：StatefulWidget要继承，还要重写State类，重写的State也要继承State
    • MyApp不需要状态处理，继承StatelessWidget组件，里面需要的MyHomePage继承StatefulWidget；
    • MyHomePage中重写createState方法为_MyHomePage；
    • _MyHomePage继承了State<MyHomePage>，并添加状态方法（按键），重写Widget构建；
  • 包资源的使用

Dart语言

  • 声明变量使用var关键字，默认为null
  • 常量final之恩那个被赋值一次，const是常量
  • Dart可以用三个’’’或者”””来包含多行字符串
  • Dart不把1视为true，只有true为true
  • 函数的可选参数用[]括起来
  • 返回整值的除法~/
  • b ??= value仅在b为null时赋值
  • ..是级联（暂作保留）
  • 函数中的get和set方法可以设置类似关联的关系
  • 泛型最常见使用，使只包含字符串

    异步支持

  • Future实例、async函数和await表达式实现异步操作
  • 处理网络请求或者文件读取，不能一直等一个线程导致阻塞。使用多线程（Java、C++）或者单线程+事件循环（JavaScript、Dart）
  • 阻塞调用和非阻塞调用：阻塞调用是结果返回之前，当前线程一直被挂起；非阻塞则是过段时间回来检查是否有结果
  • 创建Future实例，通过.then()的方式来监听Future内部执行完成时得到的结果
  • Future可以不阻塞线程，让线程继续执行，在某个操作改变自己的状态后，回调then或者errrorCatch回调
  • Future将任务添加到事件队列的队尾

  • 通过await获取异步函数Future的结果，返回的结果包装在Future中。await必须在async函数中，await声明的函数都会进入阻塞状态，等到完成计算返回相应的值

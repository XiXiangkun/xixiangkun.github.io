踩过的坑

踩过的坑

网上的教程，说什么最详细，都是骗人的

### 内容：

  • 安装Git，下一步到底
  • 安装Nodejs，下一步到底
  • 使用命令行cmd安装hexo，网上有语法（注意全局配置，添加环境变量等的问题，有npm的一个刷新命令）。
  • 一些安装操作后，使用hexo -v查看版本
  • 关联Github和自己hexo搭建的博客，注意查看网上教程就可以
  • 在根目录(hexo_try)下，右键git bash进行部署（远程部署命令：hexo c清除缓存；hexo g + hexo d）

### 注意：

  1. 使用hexo的过程中，_config.yml中的：后面，一定要加空格，否则不会报错，但是不会部署成功
  2. 去github创建仓库，与自己的用户名一致，用来远程部署仓库，_config.yml中修改
    　　deploy:
    　　　　type: git
    　　　　repo: https://github.com/XiXiangkun/XiXiangkun.github.io.git
    　 　　　branch: master
  3. hexo server用来去localhost:4000查看自己当前的结果
  4. 远程部署命令：hexo -c清除缓存；hexo -g + hexo -d
  5. 可能需要登录github的窗口提示
  6. ssh密钥的配置，网上有
  7. 登录xixiangkun.github.io就可以查看博客
  8. 安装nodejs只是为了用npm安装hexo
  9. hexo发音 hack so
  10. cd命令切换 cd /d D:\software
  11. 删除github上的仓库，点开后进去setting

1.nodejs一段时间后可能需要重新安装，通过npm install和npm install hexo-deployer-git –save完成安装
2.md文件中的花括号注释在git部署的过程中会出错，所以django的文件中花括号内容被全部删除

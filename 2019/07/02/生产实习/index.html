          • min_samples_leaf叶子节点最小的样本数量

  • 深度学习：

    1. TensorFlow流程：

      • 构建计算图
      • 执行计算图
      • NOTE：
        • 需要开启会话Session，结束时关闭会话
        • 默认会话设置
        • 创建的所有节点都会自动添加到默认图
        • 图的每次执行间，所有节点值都会被丢弃，但是变量的值不会

    2. 准备数据：

      • 导入数据
      • 数据可视化：直方图…
      • 相关性分析：热力图…

    3. TensorFlow线性回归：

      • 准备工作
      • 数据准备
      • 正规方程：（Normal Equation）求解机器学习theta = tf.matmul(tf.matmul(tf.matrix_inverse(tf.matmul(XT, X)), XT), y)计算成本函数MSE
      • 梯度下降：
        • 特征缩放：标准化、归一化
        • 手工计算梯度：
        • 使用自动微分：
        • 优化器：GradientDescentOptimizer梯度下降优化器、momentum optimizer动量优化器
      • 为训练算法提供数据（深度学习算法往往不能在一次训练中加载完成，而是每次加载batch个训练数据）：
        • Placeholder创建一个占位符节点
        • Mini-batch梯度下降
      • 保存和恢复模型：定期将检查点（checkpoint）保存起来
      • 命名作用域：处理诸如神经网络等复杂模型时，图很容易就变得杂乱而庞大。为了避免这种情况，可以创建命名作用域来将相关的节点分组。

    4. TensorBoard可视化：

      • Tensorboard可以记录与展示以下数据形式：标量 Scalars、图片 Images、音频 Audio、计算图 Graph、数据分布 Distribution、直方图 Histograms、嵌入向量 Embeddings
      • Tensorboard的可视化过程
        • 建立一个graph,想从graph中获取某些数据的信息
        • 确定要在graph中的哪些节点放置summary operations以记录信息
        • 使用 tf.summary.scalar 记录标量
        • 使用 tf.summary.histogram 记录数据的直方图
        • 使用 tf.summary.distribution 记录数据的分布图
        • 使用 tf.summary.image 记录图像数据
        • operations并不会去真的执行计算，除非run,或者它被其他的需要run的operation所依赖。
        • 特地去运行所有的summary节点。一份程序下来可能有超多这样的summary 节点，要手动一个一个去启动极其繁琐，可以使用tf.summary.merge_all去将所有summary节点合并成一个节点，只要这个节点，产生所有之前设置的summary data。
        • 使用tf.summary.FileWriter将运行后输出的数据都保存到本地磁盘
        • 运行整个程序，并在命令行输入运行tensorboard的指令，之后打开web端查看可视化的结果

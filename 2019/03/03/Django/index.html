Django基本流程

1.目录层级

  • manage 用多种方式对django交互
  • init 完成python包
  • settings 项目配置文件
  • urls URL声明
  • wsgi 与WSGI兼容的web服务器入口

2.创建项目：

  • cmd下，django-admin startproject project_1 创建项目
  • tree . /F 显示层级

3.Pycharm软件操作：

  • a)init输入两条语句

    　　安装mysql 
    import pymysql  
pymysql.install_as_MySQLdb()

  • b)settings配置databases（自己用的mysql5，root-root）

    　　default内容： 
    'ENGINE': 'django.db.backends.mysql',
'NAME': "sunck",
'USER':'root',
'PASSWORD':'root',
'HOST':'localhost',
'PORT':'3306',　

  • c)数据库操作（cmd）：

    net start mysql;   
mysql -u root -p;   
create database django_try_1;

  • d)创建应用：

    python manage.py startapp myAPP(这是名字)
    　　其中admin是站点配置；model写模型；view视图

  • e)激活应用（settings中的INSTALLED_APPS）：

    　　添加’myAPP’，INSTALLED_APPS:
    'django.contrib.admin',
'django.contrib.auth',
'django.contrib.contenttypes',
'django.contrib.sessions',
'django.contrib.messages',
'django.contrib.staticfiles',
'myAPP'

  • f)定义模型（myAPP中的models）

    　　（一个数据表定义一个模型）应用有了，数据库配置了，去找模型

    ①models添加：

    form django.db import models  
class Grades(models.Model):  
　　gname=models.CharField(max_length=20)  
　　gdate=models.DateTimeField()  
　　ggirlnum=models.IntegerField()  
　　gboynum=models.IntegerField()  
　　isDelete=models.BooleanField(default=False)
    　　BooleanField只有true和false；NullBooleanField增有NULL；
    　　一个模型类对应数据库中一张表，类里面的属性对应表中的字段；
    　　学生的所在班级属性，需要用到表关联。grade与student是一对多的关系，外键写在多的里面。不规定主键，会自动生成主键。

    ②students类中关联外键，关联grade：

      　　sgrade=models.ForeignKey("Grades",on_delete=models.CASCADE)

  • g)数据库中生成数据表

    　　生成迁移文件：Django2.1只支持mysql5.6以上的版本(project_3目录下)

      　　python manage.py makemigrations
  　　python manage.py migrate

    ①测试数据

      　　python manage.py shell

    ②然后在shell中输入：

    引入包

      　　from myApp.models import Grades,Students
  　　from django.utils import timezone
  　　from datetime import *

    查询所有数据

      　　Grades.objects.all()#类名.objects.all()

    ③添加数据（本质是创建模型类的对象实例）

      　　grade1=Grades()
  　　grade1.gname="python00"
  　　grade1.gdate=datetime(year=2019,month=3,day=8)

    ④查看数据库

    在数据库中操作：

      　　select * from myapp_grades

    　　没有数据，想把数据存进去，需要保存。回去project_3的cmd 

      　　grade1.save()

    　　查看数据会有显示

      　　Grades.objects.all()

    　　显示不方便，在models的Grades类中重定义str(self)函数：

      　　def __str__(self):
  　　    return "%s-%d-%d"%(self.gname,self.ggirlnum,self.gboynum)

    　　重新打开shell，导入包，显示all，可以看到信息
    　　查看某个对象，在插入对个对象后，shell中： 

      　　Grades.objects.get(pk=2)#primarykey

    ⑤修改数据

      　　grade1.gboynum=0
  　　grade1.save()

    ⑥删除数据

      　　grade1.delete() #物理删除

    #逻辑删除 ？？

  • h)关联对象（创建学生）

      　　grade1=Grades.objects.get(pk=1)
  　　stu=Students()
  　　……整理信息
  　　stu.sgrade=grade1
  　　stu.save()

    　　获取关联对象的集合（拿到1班所有学生） 

      　　grade1.students_set.all() #这是自带的函数，一对多的多.类名小写_set.all()
  　　创建一个学生属于一个班级:  
  　　stu3=grade1.students_set.create(sname=u'XXK',sgender=True,scontend='I am XXK',sage=13)
  　　# 不用save

  • i)启动服务器

      　　python manage.py runserver #可加ip:port
    　　#ip不写代表本机ip，port默认8000。是一个轻量级的测试网页

4.admin站点管理

　　#内容发布（负责添加、修改、删除内容）+公告访问
　　配置admin应用（settings中的INSTALLED_APPS中得到django.contrib.admin，默认已经添加）

  • a)创建管理员用户

      　　python manage.py createsuperuser
    　　账户名、邮箱、密码
      　　sunck、akun2018
    　　页面中url添加’/admin’，可以观察登录runserver

  • b)汉化

    　　去settings中的

      　　language_code='zh-Hans'
  　　time_zone='Asia/Shanghai'

  • c)管理数据表（修改admin文件）

      　　from .models import Grades,Students

  　　admin.site.register(Grades)
  　　admin.site.register(Students)

    　　去网页端测试，可以看到信息

  • d)自定义管理页面（在admin中）

    　　添加的类要放到参数里面才能用

      　　class GradesAdmin(admin.ModelAdmin):
  　　    #列表页属性
  　　    list_display=['pk','gdate','ggirlnum']#显示字段
  　　    list_filter=['gname']#过滤器，过滤条件的
  　　    search_fields=['gname']#可以按gname搜索
  　　    list_per_page=5#分页

    　　admin.site.register(Grades,GradesAdmin)

  • e)关联对象

    　　创建一个班级时，直接添加几个学生

      　　class StudentsInfo(admin.TabularInline):
  　　    model=Students #添加学生
  　　    extra=2 #添加两个学生

  　　class GradesAdmin(admin.ModelAdmin):
  　　    inlines=[StudentInfo] #

  • f)布尔值显示问题

      　　def gender(self):
  　　    if self.isDelete:
  　　        return "男"
  　　    else:
  　　        return "女"
  　　gender.short_description="性别"  #修改网页上面的显示（似乎只有函数可以）

    　　再显示调用直接就是gender

  • h)执行动作的位置问题

      　　action_on_top=True
  　　action_on_bottom=False

  • i)使用装饰器完成注册，类还是有，不过

      　　@admin.register(Students)
    　　取代
      　　admin.site.register(Students,StudentsAdmin)

5.视图的基本使用

　　django中，视图对web请求回应

  • a)在views中定义视图

    　　写好返回什么页面： 

      　　from django.http import HttpResponse  
  　　def index(request):  
  　　    return HttpResponse("sunck is here")

  • b)为视图配置url(project的urls.py)

    　　类似于一个总的对应梳理 

      　　from django.urls import path,include
  　　urlpatterns = [
  　　    path('admin/', admin.site.urls),
  　　    path('',include('myApp.urls')),
  　　]

  • c)引导myApp下创建新的urls.py匹配视图

    　　用来匹配url路径和显示的views页面

      　　from django.urls import path
  　　from . import views
  　　urlpatterns = [
  　　    path('',views.index),
  　　]

后续修改

  • d)在views中定义视图

    　　写好返回什么页面，有参数的加入

      　　from django.http import HttpResponse
  　　def index(request):
  　　    return HttpResponse("sunck is here")
  　　def detail(request,num):
  　　    return HttpResponse(["sunck is here:",num])

  • e)为视图配置url(project的urls.py)

    　　类似于一个总的对应梳理，没有变化

      　　from django.urls import path,include
  　　urlpatterns = [
  　　    path('admin/', admin.site.urls),
  　　    path('',include('myApp.urls')),
  　　]

  • f)引导myApp下创建新的urls.py匹配视图

    　　用来匹配url路径和显示的views页面
    　　尖括号用来匹配数字，可以作为参数传入views中的函数，注意正则表达式使用
    path匹配不了的用re_path

      　　from django.urls import path
  　　from . import views
  　　urlpatterns = [
  　　    path('',views.index),
  　　    path('<int:num>/',views.detail),
  　　]

6.模板的基本使用

　　模板是HTML页面，根据视图中的数据进行填充

  • a)创建模板

    　　project下创建directory，与myApp共存，里面创建myApp文件

  • b)配置模板路径

    　　修改settings文件的templates

      　　'DIRS': [os.path.join(BASE_DIR,'templates')],

    　　上面的

      　　BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
  　　base_dir代表'D:\django\project_3'，也就是manage在的路径

  • c)创建html文件（想在grades页面显示列表）

    　　在模板myapp中创建html页面
    　　模板语法： 

    　　<!DOCTYPE html>
    　　<html lang="en">
    　　<head>
    　　    <meta charset="UTF-8">
    　　    <title>班级信息</title>
    　　</head>
    　　<body>
    　　    <h1>班级信息列表</h1>
    　　    <ul>

    　　    </ul>
    　　</body>
    　　</html>

　　href的内容点击后会传送到url的后缀里面

  • d)匹配url,修改myApp\urls.py

      　　urlpatterns = [
  　　    path('',views.index),
  　　    path('<int:num>/',views.detail),
  　　    path('grades/',views.grades),
  　　]

  • e)需要对视图进行更改

      　　from .models import Grades
  　　def grades(request):
  　　    # 去模板取数据
  　　    gradesList=Grades.objects.all()
  　　    # 将数据传递给模板，模板渲染页面，渲染好的页面返回浏览器
  　　    #myApp前面不写，因为已经配合好了路径settings
  　　    #返回的是字典
  　　    return

总流程：

  • ①.在html中写模板

  • ②.定义视图

  • ③.配置url

  • ④.匹配视图

  • ⑤.视图中定义模型取数据，传给模板渲染，返回浏览器

过程中的报错

  • ①.python manage.py makemigrations总是出错。
  • ②.主要原因在于使用的mysql密码没有对应，忘记原先密码。谷歌搜索更改密码后，重新验证，port为13140。
  • ③.新创建的数据库无法生成迁移文件，原先存在的mysql可以正常迁移，删除表后报错，现改到数据库information_scheme。
  • ④.以上不算，mysql出了问题，系统数据表被不小心删除，重装后解决问题
  • ⑤.重新安装后，mysql在D:\mysql\mysql-5.7.25-winx64\bin，cmd管理员身份；django使用projct_3
  • ⑥.数据库的部分操作
      　　net start mysql
  　　mysql -u root -p
  　　use sunck
  　　show tables;
  　　desc myapp_grades;
  • ⑥.重新迁移最好把migtations中的0001_initial.py文件删除，并把数据库删除
